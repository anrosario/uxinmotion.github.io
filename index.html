01Principles of UX in Motion

Easing

Easing applies acceleration to UI elements, causing them to move like how objects do in the real world. It adds timing to animations which make them feel more natural.
02Principles of UX in Motion

Offset & Delay

The offset and delay principles establishes hierarchy among UI elements. They add timing and spacing to the elements in a group in a staggered manner to distinguish each element. They also help users identify individual functionalities of UI elements that are grouped together.
03Principles of UX in Motion

Parenting

Parenting links the properties of an object to those of another. When a parent element's properties change, so do the related properties of its child elements. For example, when a child element's position is linked to that of its parent, the child element moves in the same direction.

04Principles of UX in Motion

Transformation

Transformation helps show progression in an element by changing its shape or form. A submit button, for example, turns into a progress bar, and then into a check mark once the process is completed.

05Principles of UX in Motion

Value Change

Websites commonly contain data that are represented numerically, textually, or graphically. These data change as they are derived from real-time information. Value change shows that the data are dynamic in nature, informing users that they are interactive and may be affected to some degree. Adding motion to these representations enhances engagement with the users.
06Principles of UX in Motion

Masking

Masking is the process of showing and hiding parts of a UI element. Masking allows a change in an element's use by changing its form while retaining its identity.

07Principles of UX in Motion

Overlay

Overlay creates spatial relationships in a non-3D space like websites. It also shows hierarchy in some instances. Examples are pop-up messages displaying information or options which require interaction from the user.
08Principles of UX in Motion

Cloning

Cloning conveys relationships and narratives among UI elements. This occurs when elements originate from another element, demonstrating a clear chain of events.
09Principles of UX in Motion

Obscuration

Obscuration focuses attention to one element by hiding others through blurring or lowering the opacity of UI elements. Navigation menus typically use this approach, in which a full-screen navigation page displays on top of the current page when the hamburger icon is clicked or touched.
10Principles of UX in Motion

Parallax

Parallax creates hierarchy by showing elements that move simultaneously but at different speeds. This priniciple simulates the real world, in which objects closer to us move faster and those farther away move slower.
11Principles of UX in Motion

Dimensionality

Dimensionality creates depth in a 2D space by making elements appear as objects in the real world. This principle is commonly applied to screen transitions or animations when a user interacts with an element.
12Principles of UX in Motion

Dolly & Zoom

The dolly and zoom principles are principles applied in films which relates to camera movement.


Dolly happens when the point of view moves toward or away a UI element. Zooming, on the other hand, occurs when the size of the UI element changes without its position changing.
